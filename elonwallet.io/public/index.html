What is hardware-graded security?
‍
Hardware-graded security or hardware-based encryption offers an additional protection layer. Only a specific hardware module can read and use its encryption/signing key. This key is protected by hardware and is resistant to the majority of side-channel attacks.
‍
Why not use software-based encryption?
‍
Encryption done by software is more likely to have bugs and the key is not protected against unauthorized reading. Keys generated by the software can be stored password-encrypted. However, when an adversary has access to your system, he will be able to read the password from memory.
‍
What is confidential computing/enclave technology?
‍
An enclave is an isolated process running on an untrusted system. Isolation is achieved by encryption and integrity protection of the entire process through the CPU. Since only the CPU can read and use the encryption key, anyone with system access only sees ciphertext for the encrypted process. Computations in an enclave are done efficiently and the data is read protected and never leaves the CPU unencrypted. This is referred to as confidential computing.
‍
How is my wallet key being stored?
‍
A big problem, which arises when using crypto wallets is, where and how the wallet key is being saved. All your wallet keys will be stored encrypted by the CPU (Intel SGX sealing). Through memory encryption and hardware protection, no one can know the used encryption key and hence not your wallet key.
‍
What is the meaning of passwordless (FIDO2/WebAuthn)?
‍
With FIDO2/WebAuthn you can convince us about the knowledge of a secret, without actually revealing it to us. We don’t know the secret you actually use and any adversary also. In addition to a simpler authentication process, it is more secure. This is achieved by cryptography with modern security standards. You don’t have to remember or worry about the selection of a secure password, passwordless does this automatically. Passwordless authentication uses the magic of cryptography.
‍
What are Passkeys?
‍
It is a collection of passwordless authentication methods.
‍
Is my biometric data being sent to your servers?
‍
No, biometric data will stay on your system. See “What is the meaning of passwordless (FIDO2/WebAuthn)?”.
‍
Do ElonWallet or the cloud providers have access to my wallet?
‍
No, your keys are stored encrypted by the CPU. Due to confidential computing, your wallet keys will stay confidential, even when doing transactions. Before a transaction from your wallet will be sent, a FIDO2/WebAuthn (passwordless) authentication process against the enclave is required. As only you can authorize with for instance fingerprint or FaceID transactions, no other can do transactions.
‍
How can I be sure that ElonWallet doesn’t have a backdoor in their system?
‍
We are an open-source project and all of our code is on GitHub. Everyone, who knows GoLang and Nuxt, can verify our code. Intel SGX has a feature called remote attestation. With remote attestation, you can verify that our published code on GitHub is running in the enclave.
‍
Why should I trust ElonWallet?
‍
The short answer is, you don’t have to. Everything we do is transparent and publicly verifiable. You only have to trust in the technology (Intel SGX, FIDO 2) we use.
‍
What is a cold wallet?
‍
A cold wallet is a type of hardware module with cryptographic protection against unauthorized access. They are not connected to the internet and therefore protected against any attack done over the internet.
‍
What is a non-custodial wallet?
‍
It is a wallet, where you own its private key.
‍
